Pengertian GIT

GIT adalah singkatan dari Group Inclisive Tour, merupakan komponen penting untuk website. Secara lebih lengkap, GIT merupakan tools yang digunakan untuk pengembangan perangkat lunak dan website juga digunakan untuk mengelola versi source code program dengan cara menentukan baris serta kode yang perlu diganti atau ditambahkan. GIT dikategorikan sebagai Distributed Version Control System (DVCS) yang berarti sumber kode tidak hanya dimiliki dalam satu penyimpanan pada sebuah software. Pada Git, setiap Salinan kode dari para pengembang memiliki peran sebagai repository yang berisi riwayat lengkap setiap perubahan.

Fungsi GIT

  1. Digunakan untuk berkolaborasi
  2. Sebagai platform fleksibilitas
  3. Membantu organisir
  4. Proyek open source
  5. Menjadi backup

Fitur GIT

  1. Version Control System
  2. Branch Code
  3. Bersifat Atomic
  4. Media penyimpanan dalam Folder .git
  5. Data Model
  6. Staging Area atau Index

Perintah dasar GIT

  1. Git-Init: untuk membuat repository yang ada pada file local dan berlokasi di folder .git
  2. Git Status: untuk mengecek status repository lokal
  3. Git add: untuk perintah menambahkan file baru ke repository yang baru dipilih
  4. Git commit: untuk menyimpan apabila terjadi perubahan dan dilakukan pada repository jarak jauh, namun tidak bisa melakukan perubahan
  5. Git push: perintah yang digunakan untuk mentransfer peruubahan file ke repository jarak jauh setelah melakukan perubahan
  6. Git Checkout: perintah dasar yang digunakan untuk menukar cabang aktif dengan cabang yang dipiih
  7. Git Merge: perintah dasar yang menggabungkan cabang aktif dengan cabang yang dipilih
  8. Git Clone: perintah dasar untuk membuat Salinan repository lokal

Perbedaan GIT dan GitHub

GIT:

  • Dikelola oleh The Linux Foundation
  • Diakses secara offline
  • Tidak memakai fitur user management
  • Install software di penyimpanan lokal
  • Fokus pada version control dan code sharing
GitHub:
  • Diakuisisi oleh Microsoft
  • Diakses secara online
  • Menggunakan fitur user management
  • Di-host melalui layanan cloud
  • Fokus pada source code hosting yang terpusat
